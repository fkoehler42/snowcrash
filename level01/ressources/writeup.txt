### Changes to write here following the Kali ssh setup (WIP) ###

/!\ From here, we will sometimes use a Kali Linux virtual machine which provide some useful security tools.
To set it up, first log in as root, create a new user who belongs to sudo group, then reset the ssh service settings to default to enable remote connection easily.

`useradd -m fkoehler -G sudo -s /bin/bash`
`passwd fkoehler`
`[type new password twice]`
`update-rc.d -f ssh remove`
`update-rc.d -f ssh defaults`
`systemctl enable ssh` #auto-start ssh on boot
`service ssh restart`
`su fkoehler`
`[type password]`


- Back to SnowCrash machine, look at the file '/etc/passwd' to reveal the hashed password for the user flag01.

`cat /etc/passwd | grep flag01`

- On host machine, copy the 'passwd' file from SnowCrash to Kali in order to use the password decryption tool 'John the Ripper'

`scp -P 4242 level00@[SnowCrashIPAddress]:/etc/passwd .` ##command to update to make the copy from snowcrash to kali on host machine
